# Master Prompt: ระบบจ่ายน้ำมันอัตโนมัติด้วย QR Code

## เป้าหมาย
- สร้างระบบจ่ายน้ำมันอัตโนมัติ
- ให้ลูกค้ากำหนดจำนวนเงินที่ต้องการจ่าย
- ชำระเงินผ่าน QR Code (PromptPay/ธนาคาร)
- ตรวจสอบสถานะการจ่ายเงินผ่าน API ของ Payment Gateway
- อนุญาตจ่ายน้ำมันเมื่อชำระเงินสำเร็จ

## Flow หลัก
1. ลูกค้าเลือก/กรอกจำนวนเงินที่ต้องการจ่ายค่าน้ำมัน
2. ระบบสร้าง QR Code สำหรับยอดเงินที่ระบุ (เชื่อมกับ Payment Gateway หรือใช้ PromptPay API)
3. แสดง QR Code ให้ลูกค้าสแกนจ่าย
4. ระบบตรวจสอบสถานะการชำระเงินแบบ real-time (Polling หรือ Webhook)
5. เมื่อชำระเงินสำเร็จ ระบบแจ้งเตือนและสั่งอนุญาตให้จ่ายน้ำมัน

## เครื่องมือ/เทคโนโลยีที่ใช้ (ตัวอย่าง)
- Frontend: React + TailwindCSS + GSAP (UI/UX ลื่นไหล, Interactive)
- Backend: Node.js/Express หรือ No-Code Platform ที่รองรับ Webhook/REST API
- QR Code Generation: ไลบรารีสร้าง PromptPay QR หรือ Payment Gateway API
- Payment Status: Polling API หรือรอ Webhook แจ้งสถานะ
- Hardware Integration: Relay/IoT Controller (สั่งปลดล็อคหัวจ่ายน้ำมัน)

## ข้อมูลที่ต้องการ
- ยอดเงินที่ลูกค้าต้องการจ่าย
- ข้อมูลการชำระเงิน (Transaction ID, Status)
- สถานะหัวจ่ายน้ำมัน (อนุญาต/ไม่อนุญาต)

## Workflow (กระบวนการทำงาน)
1. ลูกค้าเลือกจำนวนเงิน → กดสร้าง QR
2. ระบบสร้างและแสดง QR code
3. ลูกค้าสแกนจ่าย → ระบบรอผลการชำระเงิน
4. เมื่อจ่ายเงินสำเร็จ → ระบบปลดล็อกหัวจ่ายน้ำมัน
5. หากไม่สำเร็จภายในเวลาที่กำหนด → ยกเลิกคำสั่ง

## เงื่อนไข/ข้อควรระวัง
- ต้องป้องกันการจ่ายเงินซ้ำ/ผิดจำนวน
- ควรมี timeout และระบบคืนเงินในกรณีจ่ายเงินแต่ไม่ได้รับน้ำมัน
- ระบบต้องรองรับการเชื่อมต่อกับอุปกรณ์ควบคุมหัวจ่ายน้ำมันอย่างปลอดภัย

## จุดเด่นที่ต้องการ
- UI/UX สวย ลื่นไหล Responsive
- ตรวจสอบสถานะเงินเข้าแบบ real-time
- ปลอดภัยและเชื่อถือได้

## ตัวอย่างการใช้งาน Prompt นี้

> 
> "ออกแบบหน้าจอ React + TailwindCSS + GSAP สำหรับขั้นตอนเลือกเงินและแสดง QR Code พร้อม polling สถานะเงินเข้า"
